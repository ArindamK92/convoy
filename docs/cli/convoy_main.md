# Combined Runner CLI (`convoy_main.py`)

Top-level parser lives in `convoy_parser.py`.

## Purpose
- Run optimal+heuristic and/or RL from one command
- Share common args (`customer-num`, `charging-stations-num`, `ev-num`, energy rate)
- Aggregate output rows into one results CSV

## Common Command
```bash
python3 convoy_main.py \
  --combined-details-csv data/combined_data_jd200_1.csv \
  --combined-dist-matrix-csv data/distance_matrix_jd200_1.csv \
  --combined-time-matrix-csv data/time_matrix_jd200_1.csv \
  --customer-num 10 \
  --charging-stations-num 3 \
  --ev-num 2 \
  --opt-rl-extra "--print-solution --save-model"
```

## Multiple Iterations + Custom Results File
```bash
python3 convoy_main.py \
  --combined-details-csv data/combined_data_jd200_1.csv \
  --combined-dist-matrix-csv data/distance_matrix_jd200_1.csv \
  --combined-time-matrix-csv data/time_matrix_jd200_1.csv \
  --customer-num 10 \
  --charging-stations-num 3 \
  --ev-num 2 \
  --iterations 10 \
  --results-file jd200_itr10.csv \
  --opt-rl-extra "--print-solution --save-model"
```

## Runner Controls
- `--only-rl`: skip optimal+heuristic
- `--only-opt-heu`: skip RL
- `--opt-rl-extra "..."`
- `--opt-heu-extra "..."`

## Meaning Of Extra Args
- `--opt-rl-extra` forwards flags to the RL runner parser.
- `--opt-heu-extra` forwards flags to the opt+heu runner parser.
- These are pass-through strings, so keep forwarded flags quoted.

Examples:
- `--opt-rl-extra "--print-solution --save-model"`
- `--opt-rl-extra "--seed 77 --test-csv data/test_instance.csv"`
- `--opt-heu-extra "--skip-optimal"`
- `--opt-heu-extra "--random-seed 123 --alpha1 1.0 --alpha2 1.0"`

## Notes
- If both runners are enabled, optimal+heuristic executes first.
- `test_instance.csv` is generated by opt+heu preprocessing and auto-used by RL unless `--test-csv` is explicitly passed in `--opt-rl-extra`.
- With `--iterations > 1`, opt/heu sampling uses an iteration-specific seed to vary sampled customers/CPs across runs.
- Optional base seed can be passed through `--opt-heu-extra="--random-seed <INT>"`.
- With `--iterations > 1`, if RL `--seed` is not explicitly passed in `--opt-rl-extra`, the seed is offset per iteration.
- For extra args, keep each extra value quoted:
- `--opt-rl-extra "--print-solution --save-model"`
